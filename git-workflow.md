# Git Workflow

---

## Основные ветки

main  
Стабильная версия проекта.

develop  
Основная ветка разработки, в которую объединяются новые функции.

feature/*  
Ветки для разработки новых функций.

Примеры:

feature/booking-system  
feature/user-authentication

bugfix/*  
Ветки для исправления ошибок.

Пример:

bugfix/login-validation

---

## Процесс работы

1. Создать новую ветку от develop

git checkout develop
git checkout -b feature/new-feature

2. Реализовать функциональность

3. Сделать коммит

git commit -m "Добавлена система бронирования"

4. Отправить ветку в репозиторий

git push origin feature/new-feature

5. Создать Pull Request

6. После проверки объединить ветку с develop

---

## Правила коммитов

Коммиты должны иметь понятное описание.

Примеры:

feat: добавлен API бронирования  
fix: исправлена ошибка авторизации  
docs: обновлена документация  
refactor: улучшена структура сервисов

---

## Pull Request

Каждый Pull Request должен содержать:

- описание изменений
- связанную задачу (если есть)
- пояснение по тестированию
